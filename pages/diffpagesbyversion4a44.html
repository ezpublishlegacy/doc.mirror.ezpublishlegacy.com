This document describes the eZ Publish features/environments and customizations supported by the cloud. Anything not mentioned in this doc is by default not supported. 

 

eZ Publish 5.1LS1 LS[1] 


  1. eZ Publish 5.1.0 out-of-the-box, no kernelhacks kernel hacks or changes (except for any service packs). This also implies no kernel overrides
  2. eZ Publish in virtual host mode (standard eZ Publish5.1LS 1 LS rewrite rules)
    1. Custom apache rewriterules must be specified in eZ cloud Cloud extension - if any
    2. HostMatch setting in site.ini must contain all production hostnames (“localhost” or any non-unique names not allowed)
  3. eZ Publish uploaded to cloud must be configured for singlemode single mode (once uploaded to the cloud, the installation will run in ezdfs eZDFS cluster mode)
    1. Since site is running in cluster mode, there are a few things in the admin interface which cannot be used as they are not cluster aware. This includes (but is not limited to)
      1. Packages management
      2. Ini editor
    2. Since the site is running in cluster mode, custom extensions needs to be cluster aware.
      1. Do for instance not save files on local file system and expect it to be available on all webservers
  4. eZ Publish must be configured in host-matching mode (uri based and others not supported)
  5. eZ Find is supported, however only with default one-core setup (no multilingual)
    1. We support custom solr configs on request (needs review)
      1. custom Custom config tarball (schema.xml, protwords.txt, stopwords.txt, synonyms.txt) to be uploaded to ftp and must be specified in ezcloud extension.
    2. Custom solr configs must be reviewed and approved by eZ Cloud team in advance. Requests might be declined due to security or technical constraints
  6. Only MySQL 5.1 is supported
    1. As a side effect, ezoracle eZ Oracle is not supported in the cloud
  7. Single database installation (using one database connection for all siteaccesses)
    1. As a side effect, ezcontentstaging eZ Content Staging is not supported in the cloud
  8. Custom Database tables are supported (as long as they are in the same DB)
  9. No database GUI available (only DB imports and exports are possible)
  10. Only PHP 5.3 supported
    1. Encrypted phar package files are not supported as there is a bug with RHEL’s version of PHP 5.3
    2. The following PHP extensions are supported:
      1. apc
      2. bz2
      3. calendar
      4. Core
      5. ctype
      6. curl
      7. date
      8. dom
      9. ereg
      10. exif
      11. fileinfo
      12. filter
      13. ftp
      14. gd
      15. gettext
      16. gmp
      17. hash
      18. iconv
      19. json
      20. libxml
      21. mbstring
      22. mcrypt
      23. mongo
      24. mysql
      25. mysqli
      26. openssl
      27. pcntl
      28. pcre
      29. PDO
      30. pdo_mysql
      31. Phar
      32. posix
      33. readline
      34. Reflection
      35. session
      36. shmop
      37. SimpleXML
      38. soap
      39. sockets
      40. SPL
      41. sqlite3
      42. standard
      43. suhosin
      44. sysvmsg
      45. sysvsem
      46. sysvshm
      47. tokenizer
      48. xml
      49. xmlreader
      50. xmlwriter
      51. xsl
      52. zlib
  11. Cronjobs must be specified in ezcloud extension
  12. Mail settings using SMTP (configured automatically, any custom settings will be overridden)
  13. Only http and https outgoing traffic is allowed and must be opened server-side by Ixonos [2]
    1. list List of hostnames / ip must be provided by client
    2. Wildcards (access to anywhere) not allowed
  14. CDN : not supported at the moment
  15. Site will run behind Varnish
    1. Varnish configuration is fixed (cannot be changed by customer)
    2. Non-logged-in users in live sites will be served by varnish
  16. Extension located in extension/ is supported
    1. Also, all templates and designs must be located in an extension (not in [docroot]/design/)
    2. Extensions must be designed to be able to work without the need for any daemons, scripts etc.
  17. Shell access is not available
    1. No scripts can be executed from command
  18. Access and error logs are not available
  19. Open office is not running as daemon in the environment, so imports provided by the ezodf extensions which depends on the daemon is not supported ( like .doc and .pdf imports ).
  20. Http supported, https not supported
  21. Sites uploaded to the eZ Cloud must work with eZ Publish’ caches enabled. This includes (but is not limited to):
    1. View cache
    2. Template cache
    3. Role cache
    4. Policy cache
    5. Design location cache
    6. Translation cache
    7. Template compile
    8. Template optimization
    9. Development mode is disabled
    10. Override cache
  22. Settings which be automatically disabled in the cloud
    1. SQL debug
    2. Redirection debug
    3. Debug output
    4. Regional settings debug
    5. Template debug
  23. Http /php settings
    1. upload_max_file = 100Mb
    2. post_max_size = 100Mb
    3. max_execution_time = 120
    4. memory_limit = 128Mb
