Known existing integration (white list):

FeatureSupportedLimitations
ConfigurationAll configuration uses standard yml configuration
  1. Siteaccess (Semantic) configuration is something that has been built on top of Symfony
    1. Standard Services does not support them
    2. They are not cached but lookup occures based on cached Symfony configuration
  2. Unit tests currently use ini files and own configuration system
Dependency injectionInjection Supported for all services
  1. See known issues (eg. missing possibility to inject Criterion)
  2. Unit tests currently use own dependnecy injection system driven by own ini system
SecurityOwn Firewall voter that uses eZ Publish user rights
  1. eZ User is not the same as a Doctrine provided user
  2. Limitations does not apply at this stage but instead on data retrieval

Not provided out of the box (grey list):

FeatureLimitations
DoctrineWe don't currently use Doctrine DBAL, options: use Zeta Database/Persistence until we switch or be ok with several database connections. If the latter you can use it like described in symfony doc by adding doctrine/orm & doctrine/doctrine-bundle via composer and installs it like any other bundle.
AnnotationsWe don't currently bundle FrameworkExtraBundle which provides support for Annotating routes/firewall rules on controllers and other features, workaround: customer can add it in composer and installs it like any other bundle.
