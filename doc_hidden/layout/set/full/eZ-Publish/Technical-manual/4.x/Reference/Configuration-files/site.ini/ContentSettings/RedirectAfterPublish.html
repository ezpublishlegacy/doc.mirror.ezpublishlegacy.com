RedirectAfterPublish

Summary

Sets the location where to redirect after publish (5.2 and higher)

Usage

RedirectAfterPublish=[value]

Description

Sets the preferred location where to redirect after publishing content.

It can contain the following values:

  • parentNode - redirect to the parent of the edited object's main node (default value)
  • node - redirect to the edited object's main node
  • rootNode - redirect to siteaccess root node (ie. content.ini/NodeSettings/rootNode )
  • uri - redirect to a specific URI (see RedirectAfterPublishUri settings)

Note: If the uri value is used, then URI must be specified in the RedirectAfterPublishUri setting.

Important: This setting is only available as of eZ Publish 5.2

Examples

Example #1

RedirectAfterPublish=parentNode

Example #2

RedirectAfterPublish=uri
RedirectAfterPublishUri=/content/dashboard
