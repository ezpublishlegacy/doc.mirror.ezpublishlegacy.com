[eZDFSClusteringSettings]

SettingSummary
DBBackend Sets which database back end to use when clustering with the eZDB File Handler.
DBBackend Sets the database back-end which should be used by the eZ DB File Handler.
DBConnectRetries This sets the permitted amount of connection retries when trying to connect to the database, when clustering with the eZDB File Handler.
DBExecuteRetries This sets the permitted amount of executer retries.
DBHost Sets which database host to use when clustering with the eZDB File Handler.
DBName This sets which database to use when clustering with the eZ DB File Handler.
DBPassword This sets which password to use for your database when clustering with the eZDB File Handler.
DBPort This setting sets which port to use when clustering with the eZDB File Handle
DBSocket Enables the use of sockets by eZ DFS FIle Handler.
DBUser This sets which username to use for your database when clustering with the eZDB File Handler.
MaxCopyRetries This setting defines the maximum number of times a DFS file is tried to be copied.
MountPointPath This setting defines the path to the NFS Mount Point, which should be used when clustering with an eZ DFS File Handler.
