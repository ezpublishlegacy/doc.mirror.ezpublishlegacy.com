LogDir

Summary

Sets the directory where eZ Publish stores the audit log files.

Usage

LogDir= directory_name

Description

This directive makes it possible to control where the audit logs should be stored. As of eZ Publish 4.7, the directory is relative to the VarDir defined in site.ini. The default value is "log/audit". The "AuditFileNames" setting can be used to control which activities that should be logged and to which files they should be logged. Note that the master "Audit" configuration directive must be enabled for auditing to work.

Refer to the “Audit trailing” section of the "Features" chapter for more information.
