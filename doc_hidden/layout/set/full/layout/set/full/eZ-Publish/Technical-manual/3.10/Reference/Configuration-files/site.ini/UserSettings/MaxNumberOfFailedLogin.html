MaxNumberOfFailedLogin

Summary

Sets the number of failed login attempts before user accounts are locked.

Usage

 
MaxNumberOfFailedLogin= number

Description

Sets the number of failed login attempts before user accounts are automatically locked by the system. Note that a locked user account differs from a disabled user account. A locked account can be unlocked by a user with administrator privileges. The default value of this setting is 0, which means that the number of invalid login attempts is unlimited (no user accounts will be locked).

Examples

Example 1

MaxNumberOfFailedLogin=5

This setting will limit the maximum number of failed login attempts for each user to 5. If a user attempts to log in 6 times with an invalid password, the account will be locked.
