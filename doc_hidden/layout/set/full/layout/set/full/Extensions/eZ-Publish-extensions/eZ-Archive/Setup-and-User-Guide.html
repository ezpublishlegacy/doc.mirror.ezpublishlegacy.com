Setup and User Guide

The archiving functionality will allow you to archive the stored objects in a Solr Index by copying or moving them from the original Solr Index to a new index, possibly governed by a dedicated Solr core.  Visit the following page for more information on setting up a multi-core Solr instance http://ez.no/doc/extensions/ez_find/2_3/advanced_configuration/using_multi_core_features.

In this version, a basic archiving script is provided which operates on Solr indexes themselves, but which does not manipulate the eZ Publish database. This should be done after the Solr archiving steps are completed and with dedicated scripts and cronjobs.
