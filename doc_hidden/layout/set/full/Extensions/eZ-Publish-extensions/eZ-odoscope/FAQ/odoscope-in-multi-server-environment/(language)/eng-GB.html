odoscope in multi server environment

Question:

“It looks like we will not be able to use odoscope because our installation uses multiple independent front-end servers and we do not have a way to aggregate/collate the logs from each of the servers. 

If you add a feature to collect data from multiple log locations in the future please let us know.”

Answer:

“The best practice in multi server environment is to set up the load balancer in the way that it forwards the request to the odoscope pixel always to a single web server. So there is no need to merge different log files.”
