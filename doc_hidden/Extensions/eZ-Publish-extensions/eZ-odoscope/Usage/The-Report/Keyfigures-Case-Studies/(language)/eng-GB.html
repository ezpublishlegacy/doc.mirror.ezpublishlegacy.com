Case Study 1: Bounce Rate

The bounce rate represents the percentage of visitors who entered the website and "bounced" (left) rather than continue viewing other pages of the website. It is given by the filter "Session Length = 1".

Let's see a Case Study for the bounce rate.

We have set the following:

  • Time Period: Last Month
  • Filtergroup: Origin - Filter(s) selected: Europe
  • Keyfigure - Bounce Rate
    • Filter: Session Length = 1
    • Setting: Active;
    • Indicator: Green to Red
    • Percentage: 0-10; 11-25; 26-100
  • Usergroup: Europe

With this configuration we will be able to analyze the bounce rate for European users on the website.

After we put the above settings on place, we will have the following information on odoscope Explorer:

eZ odoscope Keyfigure Study 1

  1. The Keyfigure Filter: Selected when the keyfigure was created;
  2. The Keyfigure Indicator and Percentage: The indicator will be red or green, according to the percentages you defined previously, when creating the keyfigure for the intervals. The percentage is always 100%, as each keyfigure is set upon ONE filter, so all the traffic will be shown on the selected filter. As you can see, the filter "1" shows 100%. This is the filter we chose for our keyfigure;
  3. The Filter(s): Manually selected - Europe;
  4. The Filter(s) Indicator and Percentage: The indicator will be red or green, according to the percentages you defined previously, when creating the keyfigure for the intervals. The percentage is calculated according to the the Keyfigure Filter and the Filtergroup Filter. In this scenario, it is quite simple
  • We can select just one Filter for the Keyfigure so, only the traffic filtered by this specific filter is considered when calculating the Keyfigure correlation. We can see in the column "Visits" that there were 13.602 visitors who had just one interaction with the website analyzed. It represents 61.13% of the overall amount of visitors in the time period selected, for Europe;
  • We have set only one Filter, so only the traffic filtered by it is considered when calculating the Filter percentage. Now, as our Keyfigure is Active, this value will be the same as shown for the Keyfigure, which in this case is 61.13%. If the Keyfigure was Inactive, the percentage would be calculated by the formula: (1-0,6113)*100, which is 38.87%.

Remember what Active and Inactive sets mean:

  • Active set: 61,13% of all visitors had 1 Interaction with the website (keyfigure filter)
  • Inactive set: 38,87% of all visitors had more than 1 one interaction (all the other filters)

Now take a look at the following scenario:

eZ odoscope Bounce Rate

We have added some more filters to our Filtergroup (Origin), and the correlation between the Keyfigure and the "Europe" filter has not changed, which means: from the total amount of visits from Europe, 61.13% bounced. As you can see, only the percentage of visits has changed, as we added more visits to the overall amount of visitors (by selecting other filters: North America, Asia, Africa, Oceania and South America).

Now, if we change the keyfigure to "Inactive", we will see the following results:

eZ odoscope Bounce Rate Inactive

These are the results for the Inactive setting, which gives you the percentages for the 'rest of the filters' - visitors that did not bounce.

Case Study 2: Lead Traffic Rate

The lead traffic rate represents the percentage of users visiting certain goals or lead pages. It is given by any filter available on the filtergroup LeadForms.

Let's see a Case Study for the lead traffic rate:

We want to know how many visitors from all segments ("All Users"), detailed by origin, did not click on any form available in our website in the last 90 days. We need to assess the effectiveness of a campaign we launched three months ago, to improve website usage.

We have set the following:

  • Time Period: Last 90 days
  • Filtergroup: Origin
  • Keyfigure - Lead Traffic Rate
    • Filter: LeadForms = Not Set
    • Setting: Active
    • Indicator: Green to Red
    • Percentage: 0-25; 26-50; 51-100
  • Usergroup: All Users

After setting this configuration up, we have the following results:

eZ odoscope Keyfigure Study 2

Data Analysis:

  • A total of 929 visitors entered the website using mobile devices and 96.56% (897 visitors) were filtered by the "Not Set" filter (Our Lead Traffic Rate keyfigure), which means that almost all our visitors did not access any of our forms;
  • We had some visitors who clicked in some of our forms, as for example:
    • Download the eZ Publish Enterprise 30 Day Trial: 12 visitors, which represents 1.29% of the overall amount of visitors;
    •  Demo Request: 10 visitors, which represents 1.08% of the overall amount of visitors;

            If you sum all percentages, you will get 100%.

  • From all our visitors, from all regions of the globe (all origins), we had:
    • 539 visitors from Europe, which represents 58.02% of the overall amount of visitors (929). From 539 visitors, 95.36% did not access any of our forms which is 513 visitors;
    • 278 visitors from North America, which represents 29.92% of the overall amount of visitors (929). From 278, 98.56% did not access any of our forms which is 274 visitors;

Hints: To see the percentage of users who clicked on a form, set the Keyfigure to Inactive. To see the accesses to other forms, change the keyfigure filter to any of the forms available on the filtergroup LeadForms. To see data for a specific segment, select or create a new Usergroup.

Case Study 3: Internal Search Rate

The internal search rate represents the percentage of visitors searching on the internal search bar of your website. It is given by any filter available on the filtergroup KeywordIntern.

Let's see a Case Study for the internal search rate regarding eZ Publish website.

We want to know the percentage of visitors that are using the internal search bar of the website:

eZ odoscope Keyfigure Study 3

We have set the following:

  • Time Period: Last 90 days
  • Filtergroup: Origin
  • Keyfigure - Internal Search Rate
    • Filter: KeywordIntern = Not Set
    • Setting: Inactive
    • Indicator: Red to Green
    • Percentage: 0-20; 21-50; 51-100
  • Usergroup: All Mobile Devices

After setting this configuration up, we have the following results:

eZ odoscope Keyfigure Study 3

Data Analysis:

  • A total of 929 visitors entered the website using mobile devices and 1.29% (897 visitors) were filtered by the "Not Set" filter (Our Internal Search Rate keyfigure), which means only a few of all our visitors used the internal Search Bar;
  • We had some visitors who did search, as for example:
    • Etna, which represents 0.11% of the overall amount of searches;

Hint: As you can see, the "KeywordIntern" has several filters, distributed through several pages (61 pages). Use the little arrows shown at the right side of  the filtergroup name to navigate between the pages and see more results.

  • From all our visitors, from all mobile devices, we had:
    • 443 visitors accessed the website through iPhone, which represents 47.69% of the overall amount of visitors (929). From 443 visitors, 1.29% used the internal search feature, which is 5 visitors; fil

Hints: To see the percentage of users who used the internal search, set the Keyfigure to Inactive. To see the searches for other words, change the keyfigure filter to any of the words available on the filtergroup KeywordIntern. To see data for a specific segment, select or create a new Usergroup.

More Examples:

  • Mobile Traffic Rate: SetUp in Filtergroup ‘Operating System’ – create keyfigure on Subgroup ‘Mobile Devices’ and set active;
  • Traffic Coming from Google: SetUp in Filtergroup ‘Referrer’ – create keyfigure on subgroup ‘Google’ and set active;
  • Direct Traffic Rate (typing the website url directly in the browser or visiting the website via bookmark): SetUp in Filtergroup ‘Referrer’ – create keyfigure on ‘Not Set’ and set active;

You can create an infinite number of combinations with filters, usergroups and keyfigures to analyze data from different time periods, according to what you have in mind and to your business needs.
