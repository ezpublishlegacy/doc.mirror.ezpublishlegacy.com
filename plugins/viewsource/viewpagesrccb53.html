To access the eZ Publish Cloud the user has to be a registered eZ community member (https://login.ez.no). This account gives a general access to the eZ Publish Cloud. If the user account is associated with one or more environments, also the access to the environments GUI, SFTP and GIT repository will be granted.

Each environment comes with an individual GIT repository and a SFTP account where all data for a deployment (eZ Publish installation) are managed. The SFTP and the GIT repository are SSH protected. Therefore the user has to provide his public SSH key, to be able to upload data to an environment. The ssh key will be assigned to his eZ community account and therefore be valid for all environments the user has access to. The user can add or alter his SSH key at https://login.ez.no.

GIT

The git repository will be the way you interact with the eZ Publish Cloud environment the most. Each environment has its own git repository. The git repository is the place where you upload the code you want to use for your eZ Publish installation. During the creation of a new Deployment the code is fetched from the git repository.

You are free to use this repository as you master server for you development or just upload revisions, which are marked as stable releases. To make an actual revision of the git repository available for a new deployment, you have to tag this revision and populate the tag to the server. In the GUI only the tags are shown and available for a new deployment.

SFTP

Each environment provides a sftp account. All users, who have uploaded their ssh-key, have access to this account. The credentials for the sftp can be found in the web GUI.

The sftp has initially two folders

  • db (database dumps)
  • var (collections of binary data)

The db-folder is intended to hold database dumps of eZ installations. If you create a dump of a deployment in your environment, it will be stored in this folder, from where you can fetch it. This folder is also the place where you can upload database dumps for your local installations to use them for a new deployment. All dumps in this folder are available to be used for a new deployment (selected in step 3 of the new deployment process).

The var-folder is intended to hold dumps of the binary content of a eZ Publish installation
